Description détaillée

Gère la grille de briques, le chargement et la sauvegarde des niveaux.

Paramètres
m_bricksVecteur d'objets Brick constituant le niveau.
m_gridStateTableau 2D représentant l'état de la grille (valeurs de score des briques).
m_activeBrickCountNombre de briques actives (destructibles) dans le niveau.
m_editorSelectedRowLigne sélectionnée en mode éditeur (logique, 0 en haut).
m_editorSelectedColColonne sélectionnée en mode éditeur (logique, 0 à gauche).
m_gridTotalWidthLargeur totale de la grille de briques (avec espacements).
m_gridTotalHeightHauteur totale de la grille de briques (avec espacements).
m_gridStartXPosition X de départ (coin supérieur gauche) de la grille.
m_gridStartYPosition Y de départ (coin supérieur gauche) de la grille.

◆ checkCollisions()

void Level::checkCollisions ( Ball & ball)
inline

Vérifie les collisions entre la balle et les briques du niveau en utilisant une méthode de balayage (swept collision).

Si une collision est détectée comme la plus proche dans la trajectoire actuelle de la balle, elle informe la balle de cette collision imminente (position, normale, index de la brique).

Paramètres
ballLa balle à vérifier.
Note
Cette méthode ne modifie pas l'état des briques ni de la balle directement, elle prépare la balle à gérer la collision dans son propre update().

◆ save()

bool Level::save ( const std::string & filename) const
inline

Sauvegarde la configuration actuelle du niveau (les valeurs de score de m_gridState) dans un fichier.

Paramètres
filenameLe nom du fichier de niveau.
Renvoie
true si la sauvegarde réussit, false sinon.
Note
Le fichier est écrasé s'il existe déjà.
Les valeurs de score sont écrites dans le même format que celui utilisé pour le chargement.
Les briques indestructibles sont représentées par Config::SCORE_INDESTRUCTIBLE.
Les briques vides/inactives sont représentées par 0.
Les briques actives sont représentées par leurs valeurs de score respectives.
